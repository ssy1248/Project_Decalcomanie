[게임 기획서] 고양이는 액체다! (Cats Are Liquid!)
1. 핵심 게임플레이 루프 (Core Gameplay Loop)
[탐색 및 이동] ➔ 가구/벽면의 텍스처 데이터 파악
      ▼
[냥로켓 대시] ➔ 목표 가구에 몸체 강타 (물리 충돌)
      ▼
[액체화 동화] ➔ 캐릭터 외형 텍스처 교체 + 얼굴 페널티 데칼 생성
      ▼
[집사 감시 피하기] ➔ 완벽한 착시로 집사의 의심 게이지 관리
      ▼
[검거 혹은 생존] ➔ 제한 시간 종료 시 고양이 승리 / 들킬 시 집사 승리
2. 핵심 시스템 디자인 (Core Systems Design)
① 플레이어(고양이) 캐릭터 매커니즘
조작감 (Slippery & Springy): * 평소 이동 시에는 물리 기반 서스펜션(Spring-Damper)과 주기적인 수직 충돌량(Micro-Hop)을 결합하여, 유연하고 리드미컬하게 통통 튀며 미끄러지는 고양이 특유의 질감을 구현합니다.

냥로켓 대시 (Body Slam):

직선 조준 대시 기능으로, 발사 시 중력을 무시하고 직선 속도 변화(VelocityChange)를 가해 목표 벽면에 박치기를 시도합니다.

② 액체화 및 카멜레온 동화 시스템 (Camouflage)
텍스처 복사 및 주입:

고양이가 가구/벽면(Wall.cs)에 대시로 부딪히는 순간, 해당 오브젝트가 가진 오리지널 텍스처 데이터를 흡수하여 고양이 메쉬 렌더러의 _BaseMap에 실시간으로 덮어씌웁니다.

얼굴 페널티 데칼 (Face Decal Penalty):

기믹: 몸통은 가구와 완벽히 동화되어 숨겨지지만, 고양이의 눈, 코, 입, 수염 무늬는 가구 표면에 선명하게 남는 페널티 시스템입니다.

기술 구현: 충돌 지점의 좌표(ContactPoint.point)에 URP Decal Projector를 동적 생성합니다.

회전 각도 정렬: 정면 충돌이 아닌 사선(비뚤어진 각도)으로 부딪겼을 때, 충돌 시점의 캐릭터 transform.up 또는 회전 상태를 복사하여 얼굴 데칼도 찌그러지거나 회전된 상태로 찍히도록 연산합니다. (술래에게 추리 단서 제공)

③ 술래(집사) AI 및 규칙 시스템
의심 게이지 및 탐지:

퇴근 후 돌아온 집사는 방 안을 순찰합니다. 고양이가 벽지와 다른 무늬로 숨어있거나, 비뚤어지게 찍힌 고양이 눈코입 데칼을 시야 앵글(FOV) 내에서 포착하면 '의심 게이지'가 상승합니다.

상호작용 (꿀밤/장난감):

집사가 의심스러운 가구를 타격하거나 조사했을 때, 실제 고양이의 콜라이더 상태를 체크하여 검거(Game Over) 혹은 오인(집사의 피로도 증가) 처리를 수행합니다.

3. 단계별 개발 로드맵 (Development Roadmap)
📦 Phase 1: 코어 플레이 구축 (물리 & 데이터 인터페이스)
고양이 모터 구현: 3인칭 백뷰 카메라 기준, WASD 입력 방향으로 유연하게 미끄러지며 통통 튀는 Rigidbody 물리 베이스 구축.

충돌 인터페이스: Wall.cs와 SlimeMotor.cs(CatMotor) 간의 Collision Event를 통한 Texture2D 데이터 송수신 아키텍처 완성.

🎨 Phase 2: 액체화 및 페널티 연출 (손맛 & 비주얼)
실시간 외형 교체: 가구 충돌 시 캐릭터 메쉬의 머티리얼 텍스처를 가구 무늬로 런타임 스왑하는 로직 구현.

얼굴 데칼 시스템: URP Decal Projector 프리팹을 활용하여 충돌 법선 벡터(Normal) 및 비뚤어진 각도를 반영한 얼굴 배치 시스템 구축.

임팩트 연출: 충돌 순간의 화면 흔들림(Camera Shake) 및 고양이 전용 "철퍼덕!" 이펙트/사운드 연동.

🧩 Phase 3: 콘텐츠 구현 (숨바꼭질 게임 루프)
동화율 연산 매처: 벽지 패턴과 고양이 텍스처의 일치율을 판정하는 트리거 시스템 개발.

집사 추적 로직: 정해진 루트를 순찰하며 눈코입 데칼을 탐지하는 기초 시야 시스템 및 타격 판정 구현.

라운드 매니저: 제한 시간, 승리/패배 팝업 UI 및 게임 플로우 제어 상태 머신 연동.

🛠 Phase 4: 폴리싱 및 시스템 완성 (패키징)
UI/UX: 메인 타이틀, 인게임 HUD(피로도/의심 게이지), 결과 창 화면 배치.

예외 처리 및 최적화: 허공 대시 시 고양이 탈진 페널티, 실시간 생성되는 데칼 오브젝트들의 오브젝트 풀링(Object Pooling)을 통한 메모리 관리.

4. 명확한 마일스톤 체크리스트 (Milestone Checklist)
□ 마일스톤 1: 고양이 이동 및 냥로켓 프로토타입 (Phase 1 완료)
[ ] 캐릭터(박스)가 식빵을 굽듯 쫀득하고 유연하게 통통 튀는 조작감이 구현되었는가?

[ ] 카메라 백뷰 기준으로 W를 누르면 카메라 정면 방향으로 직관적으로 미끄러지듯 전진하는가?

[ ] 특정 가구를 향해 조준 대시를 했을 때, 튕겨 나가지 않고 벽면에 탁! 정확히 정지하는가?

[ ] 패턴 벽/가구에 부딪혔을 때 로그창에 [가구 패턴_A 데이터 복사 완료]가 정상 출력되는가?

□ 마일스톤 2: 액체 고양이 비주얼 & 손맛 (Phase 2 완료)
[ ] 가구에 부딪혀 멈추는 순간, 고양이 몸통 색상이 방금 부딪힌 가구의 텍스처와 완벽히 일치하도록 실시간 교체되는가?

[ ] 부딪힌 위치 좌표에 고양이의 눈코입과 수염 무늬(Decal)가 밀림 현상 없이 정확하게 남아서 출력되는가?

[ ] 사선이나 비뚤어진 각도로 가구에 부딪혔을 때, 남겨지는 눈코입 데칼도 비뚤어진 각도가 그대로 적용되어 찍히는가?

[ ] 박치기하는 순간 화면 흔들림(Camera Shake)과 사운드가 조화롭게 출력되는가?

□ 마일스톤 3: 잠입 숨바꼭질 루프 가능 (Phase 3 완료)
[ ] 고양이가 벽지 무늬를 복사해 똑같은 벽에 달라붙었을 때, 완벽히 벽의 일부처럼 보이는 착시가 일어나는가?

[ ] 술래(집사)가 고양이의 눈코입 데칼을 알아채고 해당 영역을 타격했을 때, 타격 판정 및 검거(아웃) 처리가 정상 작동하는가?

[ ] 제한 시간이 끝날 때까지 집사에게 들키지 않으면 고양이의 승리, 그전에 들키면 집사의 승리 팝업이 뜨고 게임이 정지하는가?

□ 마일스톤 4: 패키징 및 릴리즈 (Phase 4 완료)
[ ] 타이틀 ➔ 인게임 ➔ 결과 창 ➔ 다시 타이틀로 이어지는 루프가 버그 없이 연결되는가?

[ ] 고양이가 맵 전역의 가구를 돌아다니며 수십 번 액체화 데칼을 생성해도 프레임 드랍이나 메모리 누수가 발생하지 않는가?

[ ] 독립된 하나의 실행 파일(.exe)로 정상 추출되어 에디터 밖에서도 조작 가능한가?
